# Security policy

## Supported version

This portfolio project supports the latest commit on `main`.

## Report a vulnerability

Please do not open a public issue for a suspected vulnerability. Contact the repository owner privately through the email listed on Bertrand Rusanganwa's GitHub profile and include reproduction steps, affected components, and potential impact. Do not include live credentials or private datasets.

## Deployment notice

The repository is designed for local evaluation and as an extensible reference architecture. Before internet-facing or business-critical deployment, add TLS, secret management, centralized identity with MFA, encryption and immutability at the storage layer, alerting, network policies, dependency scanning, and formal recovery objectives. Replace all example credentials before the first start.
